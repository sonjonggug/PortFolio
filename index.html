업무 성과

서비스와 DB, 시스템 영역에서 다양한 업무를 수행하며 성과를 내었습니다.
( ※ 기초적인 유지보수 작업은 제외 하였습니다. ※ )

Service

- 외부제공용 API 서버 구축 : 외부업체에서 사용되는 API 서버를 구축하였습니다.

- 헬스체크-오토힐링 서비스 구축 : 30분 마다 서버 프로세스 상태를 주기적으로 체크하며 프로세스가 다운되었을 시 재기동, 재기동이 이뤄졌거나 실패시 메일 발송하는 서비스를 구축하였고, 또한 DB 테이블 스페이스를 30분마다 체크하여 테이블 스페이스 용량이 임계치 이상일시 메일을 발송하는 서비스를 구축하였습니다.

- 사내 서버 접속사이트 구축: 레지스트리 편집기를 조작하여 각 서버에 로그인 절차없이 putty , fileZila 등 자동 로그인을 지원하는 사내 서버 접속 사이트를 구축하였습니다. 또한 보안을 위해 정해진 IP에서만 사용 가능하게끔 처리하였습니다.

- 사내 GitLab 구축 및 주기적 백업: Docker를 활용하여 사내 GitLab을 구축하였으며, 주기적으로 백업을 수행하였습니다.

- 프로시저 스프링배치로 변환 및 젠킨스를 활용한 스케쥴링 자동화 : 기존 프로시저를 스프링배치로 변환 하였으며, 젠킨스를 활용하여 스케쥴링을 자동화하였습니다.

- 주기적 외부 API 호출 및 통계: Scheduler를 활용하여 주기적으로 외부 업체 API를 호출하고 데이터를 수집 및 통계 작업을 하였습니다.

DB

- 프로시저 작업 : DB 재부팅, 장애체크, 장비 삭제 등 다양한 프로시저 작업을 수행하였습니다.

- 다양한 통계 쿼리 작업 : DB에서 다양한 통계 쿼리를 작성하였습니다.

- Tibero → MariaDB 마이그레이션 작업 : 기존에 사용하던 Tibero를 MariaDB로 마이그레이션 하였습니다.

- 폐쇄망 MariaDB 구축 (Rocky 8): 폐쇄망에서 사용할 MariaDB를 구축하였습니다.

- MariaDB 성능 튜닝 : 업체 서버 사양에 맞춰 DB 메모리 사이즈 및 커넥션 갯수를 조정하였습니다.

- MariaDB 이중화 : Master - Slave 형식인 MariaDB Replication 작업을 진행중에 있습니다.

System

- 주기적 로그 백업 및 삭제 자동화: 주기적으로 로그를 백업하고 삭제하는 자동화 프로세스를 구축하였습니다.

- SSL : SSL 인증서 교체작업을 진행하였습니다.

- 클라우드 : 클라우드 서비스 및 자체 구축형 서비스에서 작업을 진행하였습니다.

업무 결과

  • 정해진 규격에 맞춰 API 서버를 설계 및 구축하면서 업체와의 협업성을 높이고, 원활한 데이터 연동과 업무 처리 효율을 높일 수 있었습니다.
  • 헬스체크-오토힐링 서비스를 구축하여 서버 프로세스 상태 및 DB 테이블 스페이스 상황을 주기적으로 모니터링하고 이에 대한 빠른 대응이 가능해졌습니다. 이로 인해 시스템 다운 타임이 감소하고 장애 대응 시간이 단축되어 시스템 안정성과 가용성이 향상되었습니다.
  • 기존에는 서버 접속시 로그인 절차를 거쳐야 했으나, 자동 로그인을 지원하는 사내 서버 접속 사이트 구축으로 인해 작업 효율성이 대폭 향상되었습니다.
  • 기존 SVN 저장소에서 GitLab으로 이관하여 다양한 브랜치 전략을 사용할 수 있었습니다.
  • read,process,write 등으로 나뉘어진 스프링 배치로 변환하여 유지보수가 편해지고 확장성이 높아져 기존의 프로시저와 비교하여 유연한 개발 및 운영에서 큰 효과가 있었습니다.
  • 외부 업체와의 협업에서 필요한 데이터를 정확하게 수집하고 분석할 수 있게 되어 업무 효율성이 향상되었습니다.
